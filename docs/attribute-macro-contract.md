# 属性宏定义契约

## 强制规则

重要：新增属性或扩展属性算法前，必须先更新 `src/config/attribute-macros.json`。前端 UI、默认值、本地模拟器、后端模拟器、测试与文档都必须复用同一份属性宏定义，禁止各处重复硬编码属性口径。

## 适用范围

- `units[*].stats` 下所有“数值型、可编辑、可校验”的单位属性，统一归 `attribute-macros.json`
- 这些字段的标签、默认值、下限、步进和属性角色映射，都必须从这份定义派生

## 不属于这份宏的内容

- 枚举或文本类型字段，例如：
  - `name`
  - `position`
  - `attackElement`
  - `protectionType`
- 只读派生指标，例如：
  - 有效生命
  - 有效攻击
  - 有效防御

## 当前要求

- `src/config/attribute-macros.json` 是属性系统唯一的源头定义。
- 前端页面字段顺序、标签、默认值、输入下限与步进，必须从这份定义派生。
- 如需保留阵营差异化默认值，例如红方速度 `10`、蓝方速度 `9`，也必须写回这份定义，例如通过 `teamDefaults` 描述，而不是散落在业务代码里硬编码。
- 前端本地模拟器访问属性时，必须通过属性角色映射取值，不能直接散落硬编码字段。
- Python 后端模拟器访问属性时，必须通过同一份 JSON 派生出的角色映射取值。
- 百分比类字段在中文标签中统一追加 `%`，避免 UI 与日志口径不一致。
- Smoke test、示例输入、README 规则说明变更时，必须同步核对这份定义。

## 扩展流程

1. 先修改 `src/config/attribute-macros.json`。
2. 再修改依赖该属性的算法与 UI。
3. 如有新的数值角色，再同步调整前后端属性角色映射。
4. 最后更新测试与文档，确认前后端结果一致。
