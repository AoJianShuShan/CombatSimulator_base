# N 场战斗摘要设计与落地记录

## 文档目的

这份文档用于记录 “N 场战斗最终简要结果输出” 功能的设计边界、实现口径与落地状态，避免前后端、领域模型与 UI 后续迭代时口径漂移。

## 当前结论

- 保留现有单场模拟链路不变。
- 新增独立的批量统计链路，不污染单场结果模型。
- 页面采用“结果视图切换”，而不是新增独立整页。
- `N` 属于执行参数，不属于单场战斗规则，不进入 `BattleConfig`；当前输入范围为 `1..5000` 的整数。

## 目标

1. 支持对同一份战斗配置执行 N 场模拟，并输出最终摘要统计。
2. 支持前端本地模式与后端 API 模式，两者结果口径一致。
3. 保留现有单场模式的事件回放、当前事件和事件日志能力。
4. 将页面复杂度控制在可维护范围内，不让单场 UI 与多场统计 UI 相互污染。

## 非目标

1. 本期不输出 N 场完整事件流。
2. 本期不实现批量样本战斗回放。
3. 本期不新增独立路由或独立页面。
4. 本期不修改现有属性宏和单场战斗参数宏的定义方式。

## 设计边界

### 领域边界

- `BattleInput` 继续只描述一场战斗。
- 批量执行请求单独建模，不把 `count` 塞进 `BattleInput.battle`。
- 单场结果与批量摘要结果分开定义，不混用同一个结果类型。

### 前端边界

- 配置区继续共用一套 `draft`。
- `模拟场次` 仍属于执行参数，不进入 `BattleConfig`，并继续放在共用参数区内；当前落地位置是在多场统计视图下的参数区末尾。
- 结果区新增视图切换：
  - 单场模拟
  - 多场统计
- 单场模式展示：
  - 单场摘要
  - 战斗回放
  - 当前事件
  - 事件日志
- 多场统计模式展示：
  - 参数区末尾的模拟场次输入
  - 统计摘要卡片

### 后端边界

- 保持 `/simulate` 语义不变。
- 新增独立批量接口 `/simulate-batch`。
- 批量接口默认只返回摘要，不返回完整事件流。

## Seed 策略

- 批量模式使用一个 `baseSeed` 作为根种子。
- 第 `i` 场战斗使用确定性派生函数生成子种子。
- 前后端必须使用完全一致的派生规则。
- 要求：
  - 可复现
  - 同样输入得到同样摘要
  - 不依赖运行时随机状态

## 风险点

1. `app.ts` 目前默认 `state.result` 一定带 `events`，如果结果模型不拆分，UI 会快速失控。
2. 如果把 `count` 放进 `BattleConfig`，会破坏现有战斗参数语义边界。
3. 如果批量模式复制一套简化战斗逻辑，后续规则修改会出现单场与批量结果不一致。
4. 如果批量结果返回完整事件流，接口体积和前端渲染复杂度都会失控。

## 验收标准

1. 单场模式现有能力不回退。
2. 多场统计模式可输入 `1..5000` 的整数场次并成功输出摘要结果。
3. 同一输入和同一 `baseSeed` 下，多次运行结果完全一致。
4. 前端本地模式与后端 API 模式返回相同摘要。
5. 页面结构清晰，不出现单场控件与多场统计控件混杂堆叠。

## 当前落地补充

- 前端结果区已支持 `单场模拟 / 多场统计` 切换。
- 前端本地模式与后端 API 模式都支持批量统计。
- `模拟场次` 当前仍放在整场战斗参数区域内，并只在 `多场统计` 视图下显示，不会挤动单场/敏感性分析的公共参数顺序。
- 导出结果时：
  - 单场模式导出 `battle-result.json`
  - 多场统计导出 `battle-batch-summary.json`
- 摘要结果当前重点展示：
  - 胜场与胜率
  - 平局与平局率
  - 回合制模式下的平均/最短/最长回合
  - 即时制模式下的平均/最短/最长完成时间
  - 终局净优势
  - 获胜场次总剩余血量比例
