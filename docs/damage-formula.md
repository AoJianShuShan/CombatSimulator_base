# 战斗伤害公式说明（策划核对版）

这份文档不用代码术语，按策划视角描述当前真实实现。

## 一句话版本

当前伤害结算逻辑可以理解为：

- 先看当前使用的是“Arpg即时制（时间）”还是“回合制速度高者先手”
- 先看这次攻击有没有打中
- 打中了以后，先用攻击和防御算出一个“基础伤害起点”
- 再依次吃护甲、暴击、爆头、元素克制、各种增伤减伤乘区
- 最后四舍五入取整
- 只要这次攻击命中了，最终伤害不会低于“最小伤害”

## 当前结算顺序

### 0. 先确定行动结算模式

当前支持两种模式：

- `Arpg即时制（时间）`
- `回合制速度高者先手`

默认模式是 `Arpg即时制（时间）`。

两种模式的差异只在“同一时间点谁能出手、谁先结算、什么时候扣血判死”这一层；单次命中和单次伤害公式本身是一致的。

#### Arpg即时制（时间）

- 每次先找到当前最早需要处理的时间点
- 这个时间点开始时，双方所有还能开火的单位，都算本次可以出手
- 每个单位的目标选择和伤害计算，都基于这个时间点开始时的战场状态
- 全部攻击算完以后，再统一扣血
- 所以允许出现“双方互相打死”
- 也允许出现“双方同归于尽”

#### 回合制速度高者先手

- 每次先找到当前最早需要处理的时间点
- 这个时间点内，按速度决定行动顺序
- 先行动的单位可以先打死人
- 后行动单位如果已经被击杀，就不会再出手
- `速度` 只在这个模式下决定同一时间点的先后手

### 0.1 射速、弹匣和换弹

- `射速（发/min）` 决定两次攻击之间的冷却时间：`60000 / 射速`
- 每次普通攻击消耗 `1` 发弹药
- 开局弹匣默认装满
- 技能释放和普通攻击都算一次行动，但技能动作本身不消耗弹药
- 如果本次普通攻击把弹匣打空，则立刻进入换弹
- `换弹动作时间（ms）` 决定换弹结束时间
- 只有“攻击冷却结束”且“换弹完成”同时满足，单位才会再次出手
- `最大战斗时长（ms）` 会限制时间轴继续推进；超过上限的后续时间点不会再被处理；若此时双方都还存活，则结果按平局处理

### 0.2 技能释放条件

#### 回合制速度高者先手

- 单位开局默认立刻可以放一次技能
- `回合CD` 只在这个模式下生效
- 只要当前轮次已经达到技能可用轮次，这次行动就会直接改成“释放技能”
- 放完技能后，会重新进入冷却轮次

策划语义：

- 技能不是额外插队动作，而是“替代这次本来要打出去的普通攻击”
- `回合CD = 0` 就意味着单位每次轮到自己行动时都能继续放技能

#### Arpg即时制（时间）

- 整场战斗参数里的 `初始怒气值` 只在这个模式下有效
- 单位怒气会按 `怒气恢复速度（点/s）` 随真实时间持续增长
- 怒气满 `100` 时，这次行动改为释放技能
- 放完技能后怒气会立刻清零

策划语义：

- 怒气不会单独创造一个额外动作点
- 而是在“这次本来就轮到你出手”的时候，决定它是普通攻击还是技能攻击

### 1. 先判定是否命中

- 攻击方的命中，减去目标的闪避，得到这次攻击的实际命中率
- 如果结果小于 0，就按 0 算
- 如果结果大于 100，就按 100 算
- 没有命中时，本次攻击直接结束，伤害为 0

策划语义：

- `命中高，闪避低`，更容易打中
- `闪避高到超过命中`，那就完全打不中
- 命中率再高，也不会超过 100%

## 2. 先把面板值换算成“有效属性”

当前会先把基础值和百分比加成合并，然后四舍五入成整数。

- 有效生命 = `最大生命` 叠上 `生命%`
- 有效攻击 = `攻击` 叠上 `攻击%`
- 有效防御 = `防御` 叠上 `防御%`

边界处理：

- 有效生命最低不会低于 `1`
- 有效攻击最低不会低于 `0`
- 有效防御最低不会低于 `0`

策划语义：

- 百分比加成不是单独一层乘区，而是先并入基础属性，再参与后续计算
- 生命、攻击、防御这三项都会先算“有效值”，战斗里真正拿来算的是有效值，不是面板原值

## 3. 先算“基础伤害起点”

当前先用：

- 攻击方的有效攻击
- 减去目标的有效防御

得到一个原始伤害差值。

然后和“最小伤害”比较，取较大值，作为基础伤害起点。

策划语义：

- 如果攻击明显高于防御，就按 `攻击 - 防御` 起算
- 如果攻击不够破防，或者差值太低，就直接按“最小伤害”起算

日志里的 `（不破防）`，当前判断规则是：

- 只要 `有效攻击 - 有效防御 < 最小伤害`
- 就标记为 `（不破防）`

注意：

- 这个“不破防”只看攻击和防御这一步
- 不看后面的护甲减伤
- 也不看后面的暴击、爆头、元素和其他增伤减伤

## 4. 护甲和穿甲怎么生效

护甲不是直接减固定数值伤害，而是先换算成一个“减伤率”。

先看：

- `目标护甲 - 攻击方穿甲`

如果结果小于 0，就按 0 算，也就是：

- 穿甲超过护甲时，这层护甲减伤直接失效

如果还有剩余护甲，则按下面的语义计算护甲减伤：

- 剩余护甲越高，减伤越高
- 但减伤不会无限增长
- 会受到一个“护甲减伤上限%”限制

当前实现口径：

- 护甲公式参数 `a` 默认是 `200`
- 护甲减伤上限默认是 `75%`

策划语义：

- 护甲的收益是递减型的，不是线性硬减
- 穿甲只抵消护甲，不会额外转成增伤
- 护甲减伤再高，也不会超过配置里的减伤上限

## 5. 暴击和爆头是两次独立判定

当前实现里：

- 先单独判定一次暴击
- 再单独判定一次爆头

两者互不冲突，可以同时触发。

策划语义：

- 爆头不会抢掉暴击机会
- 暴击也不会顶掉爆头
- 如果两次都触发，那么两层倍率会同时乘进去

## 6. 元素克制怎么生效

当前每个单位有：

- 攻击元素类型
- 防护类型

元素关系只看这两个字段的组合。

当前规则：

- 只要攻击元素是“无”，或者防护类型是“无”，那这层直接按 `1 倍`，也就是不参与克制
- 如果攻击方克制目标，就吃“元素克制造伤%”
- 如果攻击方被目标克制，就吃“元素被克造伤%”
- 其他情况按 `1 倍`

当前默认关系：

- 物理克制隔热甲，被重甲克制
- 火焰克制绝缘甲，被隔热甲克制
- 电磁克制生化甲，被绝缘甲克制
- 腐蚀克制重甲，被生化甲克制

当前默认倍率：

- 克制造伤：`120%`
- 被克造伤：`80%`

## 7. 其他增伤减伤怎么生效

当前还有这些独立乘区：

- 玩法场景增伤
- 英雄职业增伤
- 技能类型增伤
- 技能倍率
- 输出增幅 / 输出衰减
- 承伤加深 / 承伤减免
- 最终增伤 / 最终减伤

策划语义分别是：

- `玩法场景增伤`：攻击方在当前玩法场景下的额外伤害系数
- `英雄职业增伤`：攻击方职业对本次伤害的额外系数
- `技能类型增伤`：技能类别对应的额外系数
- `技能倍率`：默认值 `200%`；只在本次动作为“释放技能”时才会乘进去；普通攻击固定按 `1 倍`
- `输出增幅 - 输出衰减`：这是攻击方自己的输出乘区
- `承伤加深 - 承伤减免`：这是目标身上的承伤乘区
- `最终增伤 - 最终减伤`：最后一层总乘区

这里有一个重要口径：

- `输出增幅 - 输出衰减`
- `承伤加深 - 承伤减免`
- `最终增伤 - 最终减伤`

这三组在相减后，如果结果小于 `-100%`，不会变成负伤害，而是直接按 `0 倍` 处理。

也就是说：

- 这些乘区最低只能把伤害压到 `0 倍`
- 不能把伤害压成负数，也不会转成治疗

## 8. 最终伤害怎么落地

上面的所有结果会连乘，得到一个“小数伤害值”。

然后当前实现会：

1. 对这个小数做四舍五入
2. 再和“最小伤害”比较，取较大值

所以当前真实口径是：

- 只要这次攻击命中了
- 最终伤害至少等于“最小伤害”

这个点非常关键。

因为它意味着：

- 即使护甲很高
- 即使被元素克制
- 即使输出衰减很多
- 即使最终减伤很多
- 即使某个乘区刚好压成 `0 倍`

只要这次攻击判定为命中，最后仍然至少扣掉 `最小伤害`

## 9. 扣血规则

在 `回合制速度高者先手` 模式下：

- 当前生命不会低于 0
- 生命降到 0 就死亡

在 `Arpg即时制（时间）` 模式下：

- 先把这个时间点所有命中的伤害计算完
- 再把这些伤害统一结算到各自目标身上
- 统一扣血后再判定是否死亡
- 所以同一时间点内允许出现双方同时被打死

策划语义：

- 不会出现负血
- 也不会出现“被打死后还有负数残血”这种显示

## 当前边界处理汇总

这是最容易和策划预期不一致的部分，我单独列出来。

### 1. 最小伤害是“最终伤害保底”

不是“基础伤害保底”，而是“命中后的最终伤害保底”。

也就是：

- 先算完所有乘区
- 再和最小伤害比较
- 取较大值

如果策划预期是：

- “最小伤害只在攻击减防御这一步生效，后面还能被护甲或减伤压到 0”

那当前实现和这个预期是不一致的。

### 2. 概率类属性大多可以填超过 100，但实战按 100 算；护甲减伤上限会直接做输入校验

当前这些字段：

- 命中
- 闪避参与后的实际命中率
- 暴击率
- 爆头率

在真正进入公式前，都会被截到 `0 ~ 100`。

所以例如：

- 暴击率填 `150%`
- 爆头率填 `300%`

当前不会报错，但实战就是按 `100%` 算。

但 `护甲减伤上限%` 不属于这类“可超填再截断”的字段，它在输入阶段就要求 `0 ~ 100`，超出会直接报错。

### 3. 百分比减基础属性最低能压到 0，生命最低保留 1

例如：

- 攻击% = `-100%`，那有效攻击会变成 `0`
- 防御% = `-100%`，那有效防御会变成 `0`
- 生命% = `-100%`，那有效生命不会变成 `0`，而是保底 `1`

### 4. 穿甲不会把护甲打成负数

如果：

- 穿甲 >= 护甲

那护甲减伤直接视为 `0`

不会出现：

- 负护甲
- 因为“超额穿甲”再额外增伤

### 5. 三个差值乘区不会变成负数

下面三组：

- 输出增幅 - 输出衰减
- 承伤加深 - 承伤减免
- 最终增伤 - 最终减伤

差值再低，最低也只会变成 `0 倍`，不会出现负伤害。

### 6. 技能相关输入边界

- `初始怒气值` 在输入阶段就要求 `0 ~ 100`
- `回合CD` 最低是 `0`
- `怒气恢复速度` 最低是 `0`
- `技能倍率 = 0` 时，如果这次动作刚好是技能，技能乘区会变成 `0 倍`，但最终伤害仍会吃“最小伤害保底”

## 给策划核对时，当前仍建议重点确认的 4 个问题

1. 最小伤害是否真的要做成“最终命中伤害保底”？
2. 暴击率、爆头率、命中率这类字段，是否允许输入大于 `100%`，还是应该直接限制到 `100%`？
3. 护甲减伤是否只抵消伤害，不允许“超额穿甲”转成额外增伤？
4. 输出乘区、承伤乘区、最终乘区在差值为负时，是否应该直接按 `0 倍` 截断？

## 一段可以直接发给策划的话

当前版本的伤害逻辑，可以理解为：

“先看这次轮到单位出手时，当前动作到底是普通攻击还是释放技能。动作一旦确定，就先判定是否命中。命中后，先用攻击方有效攻击减目标有效防御，得到基础伤害起点，最低不会低于最小伤害。然后再依次乘上护甲减伤、暴击、爆头、元素克制、玩法场景增伤、英雄职业增伤、技能类型增伤、技能倍率、输出乘区、承伤乘区、最终增减伤乘区。这里的技能倍率只有在释放技能时才会乘进去，普通攻击固定按 1 倍。全部乘完后四舍五入取整，并再次和最小伤害比较，确保只要命中，最终伤害至少等于最小伤害。”  

如果策划要核口径，最需要确认的是：

- 最小伤害是不是要做成最终保底
- 概率类属性是否允许填超过 100%
- 三个差值乘区是否需要允许低于 0 倍
