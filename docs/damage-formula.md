# 战斗伤害公式说明（策划核对版）

这份文档不用代码术语，按策划视角描述当前真实实现。

## 一句话版本

当前伤害结算逻辑可以理解为：

- 先看这次攻击有没有打中
- 打中了以后，先用攻击和防御算出一个“基础伤害起点”
- 再依次吃护甲、暴击、爆头、元素克制、各种增伤减伤乘区
- 最后四舍五入取整
- 只要这次攻击命中了，最终伤害不会低于“最小伤害”

## 当前结算顺序

### 1. 先判定是否命中

- 攻击方的命中，减去目标的闪避，得到这次攻击的实际命中率
- 如果结果小于 0，就按 0 算
- 如果结果大于 100，就按 100 算
- 没有命中时，本次攻击直接结束，伤害为 0

策划语义：

- `命中高，闪避低`，更容易打中
- `闪避高到超过命中`，那就完全打不中
- 命中率再高，也不会超过 100%

## 2. 先把面板值换算成“有效属性”

当前会先把基础值和百分比加成合并，然后四舍五入成整数。

- 有效生命 = `最大生命` 叠上 `生命%`
- 有效攻击 = `攻击` 叠上 `攻击%`
- 有效防御 = `防御` 叠上 `防御%`

边界处理：

- 有效生命最低不会低于 `1`
- 有效攻击最低不会低于 `0`
- 有效防御最低不会低于 `0`

策划语义：

- 百分比加成不是单独一层乘区，而是先并入基础属性，再参与后续计算
- 生命、攻击、防御这三项都会先算“有效值”，战斗里真正拿来算的是有效值，不是面板原值

## 3. 先算“基础伤害起点”

当前先用：

- 攻击方的有效攻击
- 减去目标的有效防御

得到一个原始伤害差值。

然后和“最小伤害”比较，取较大值，作为基础伤害起点。

策划语义：

- 如果攻击明显高于防御，就按 `攻击 - 防御` 起算
- 如果攻击不够破防，或者差值太低，就直接按“最小伤害”起算

日志里的 `（不破防）`，当前判断规则是：

- 只要 `有效攻击 - 有效防御 < 最小伤害`
- 就标记为 `（不破防）`

注意：

- 这个“不破防”只看攻击和防御这一步
- 不看后面的护甲减伤
- 也不看后面的暴击、爆头、元素和其他增伤减伤

## 4. 护甲和穿甲怎么生效

护甲不是直接减固定数值伤害，而是先换算成一个“减伤率”。

先看：

- `目标护甲 - 攻击方穿甲`

如果结果小于 0，就按 0 算，也就是：

- 穿甲超过护甲时，这层护甲减伤直接失效

如果还有剩余护甲，则按下面的语义计算护甲减伤：

- 剩余护甲越高，减伤越高
- 但减伤不会无限增长
- 会受到一个“护甲减伤上限%”限制

当前实现口径：

- 护甲公式参数 `a` 默认是 `200`
- 护甲减伤上限默认是 `75%`

策划语义：

- 护甲的收益是递减型的，不是线性硬减
- 穿甲只抵消护甲，不会额外转成增伤
- 护甲减伤再高，也不会超过配置里的减伤上限

## 5. 暴击和爆头是两次独立判定

当前实现里：

- 先单独判定一次暴击
- 再单独判定一次爆头

两者互不冲突，可以同时触发。

策划语义：

- 爆头不会抢掉暴击机会
- 暴击也不会顶掉爆头
- 如果两次都触发，那么两层倍率会同时乘进去

## 6. 元素克制怎么生效

当前每个单位有：

- 攻击元素类型
- 防护类型

元素关系只看这两个字段的组合。

当前规则：

- 只要攻击元素是“无”，或者防护类型是“无”，那这层直接按 `1 倍`，也就是不参与克制
- 如果攻击方克制目标，就吃“元素克制造伤%”
- 如果攻击方被目标克制，就吃“元素被克造伤%”
- 其他情况按 `1 倍`

当前默认关系：

- 物理克制隔热甲，被重甲克制
- 火焰克制绝缘甲，被隔热甲克制
- 电磁克制生化甲，被绝缘甲克制
- 腐蚀克制重甲，被生化甲克制

当前默认倍率：

- 克制造伤：`120%`
- 被克造伤：`80%`

## 7. 其他增伤减伤怎么生效

当前还有这些独立乘区：

- 玩法场景增伤
- 英雄职业增伤
- 技能类型增伤
- 技能倍率
- 输出增幅 / 输出衰减
- 承伤加深 / 承伤减免
- 最终增伤 / 最终减伤

策划语义分别是：

- `玩法场景增伤`：攻击方在当前玩法场景下的额外伤害系数
- `英雄职业增伤`：攻击方职业对本次伤害的额外系数
- `技能类型增伤`：技能类别对应的额外系数
- `技能倍率`：这次出手本身的技能倍率
- `输出增幅 - 输出衰减`：这是攻击方自己的输出乘区
- `承伤加深 - 承伤减免`：这是目标身上的承伤乘区
- `最终增伤 - 最终减伤`：最后一层总乘区

这里有一个重要口径：

- `输出增幅 - 输出衰减`
- `承伤加深 - 承伤减免`
- `最终增伤 - 最终减伤`

这三组在相减后，如果结果小于 `-100%`，不会变成负伤害，而是直接按 `0 倍` 处理。

也就是说：

- 这些乘区最低只能把伤害压到 `0 倍`
- 不能把伤害压成负数，也不会转成治疗

## 8. 最终伤害怎么落地

上面的所有结果会连乘，得到一个“小数伤害值”。

然后当前实现会：

1. 对这个小数做四舍五入
2. 再和“最小伤害”比较，取较大值

所以当前真实口径是：

- 只要这次攻击命中了
- 最终伤害至少等于“最小伤害”

这个点非常关键。

因为它意味着：

- 即使护甲很高
- 即使被元素克制
- 即使输出衰减很多
- 即使最终减伤很多
- 即使某个乘区刚好压成 `0 倍`

只要这次攻击判定为命中，最后仍然至少扣掉 `最小伤害`

## 9. 扣血规则

目标掉血后：

- 当前生命不会低于 0
- 生命降到 0 就死亡

策划语义：

- 不会出现负血
- 也不会出现“被打死后还有负数残血”这种显示

## 当前边界处理汇总

这是最容易和策划预期不一致的部分，我单独列出来。

### 1. 最小伤害是“最终伤害保底”

不是“基础伤害保底”，而是“命中后的最终伤害保底”。

也就是：

- 先算完所有乘区
- 再和最小伤害比较
- 取较大值

如果策划预期是：

- “最小伤害只在攻击减防御这一步生效，后面还能被护甲或减伤压到 0”

那当前实现和这个预期是不一致的。

### 2. 概率类属性可以填超过 100，但实战按 100 算

当前这些字段：

- 命中
- 闪避参与后的实际命中率
- 暴击率
- 爆头率
- 护甲减伤上限

在真正进入公式前，都会被截到 `0 ~ 100`。

所以例如：

- 暴击率填 `150%`
- 爆头率填 `300%`

当前不会报错，但实战就是按 `100%` 算。

### 3. 百分比减基础属性最低能压到 0，生命最低保留 1

例如：

- 攻击% = `-100%`，那有效攻击会变成 `0`
- 防御% = `-100%`，那有效防御会变成 `0`
- 生命% = `-100%`，那有效生命不会变成 `0`，而是保底 `1`

### 4. 穿甲不会把护甲打成负数

如果：

- 穿甲 >= 护甲

那护甲减伤直接视为 `0`

不会出现：

- 负护甲
- 因为“超额穿甲”再额外增伤

### 5. 三个差值乘区不会变成负数

下面三组：

- 输出增幅 - 输出衰减
- 承伤加深 - 承伤减免
- 最终增伤 - 最终减伤

差值再低，最低也只会变成 `0 倍`，不会出现负伤害。

## 给策划核对时，最建议重点确认的 4 个问题

1. 最小伤害是否真的要做成“最终命中伤害保底”？
2. 暴击率、爆头率、命中率这类字段，是否允许输入大于 `100%`，还是应该直接限制到 `100%`？
3. 护甲减伤是否只抵消伤害，不允许“超额穿甲”转成额外增伤？
4. 输出乘区、承伤乘区、最终乘区在差值为负时，是否应该直接按 `0 倍` 截断？

## 一段可以直接发给策划的话

当前版本的伤害逻辑，可以理解为：

“攻击先判定是否命中。命中后，先用攻击方有效攻击减目标有效防御，得到基础伤害起点，最低不会低于最小伤害。然后再依次乘上护甲减伤、暴击、爆头、元素克制、玩法场景增伤、英雄职业增伤、技能类型增伤、技能倍率、输出乘区、承伤乘区、最终增减伤乘区。全部乘完后四舍五入取整，并再次和最小伤害比较，确保只要命中，最终伤害至少等于最小伤害。”  

如果策划要核口径，最需要确认的是：

- 最小伤害是不是要做成最终保底
- 概率类属性是否允许填超过 100%
- 三个差值乘区是否需要允许低于 0 倍
